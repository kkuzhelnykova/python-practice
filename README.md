## My Process for LeetCode

1. Read the problem twice — no code yet
2. Explain the solution out loud in plain English
3. If stuck after 15 min — use the hint, not the solution
4. Read the approach explanation only if still stuck
5. Write the code myself based on the approach
6. Only look up exact syntax when needed — this is not cheating
